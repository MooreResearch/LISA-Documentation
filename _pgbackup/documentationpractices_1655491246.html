Documentation Practices

This page describes the process of handling documentation on the website. 

Step 1: Download the current "InProgress" version of the website from the Box folder.

Step 2: Make the necessary edits to the pages you are tasked with updating.

Step 3: Test things by viewing in a browser to make sure you haven't broken anything.

Step 4: When you think you have a good version, upload the changed files to the "InProgress" folder.

Step 5: Document what you have changed and when in the "changes.html" file. Add your item as an unordered list item at the top of the list, beginning with the date and time in "YYYY-MM-DD HH:MM pm", followed by the changes you have made.

Step 6: Send any changed file and the "changes" file to Tom Moore for posting online.

Step 7: Every week or so, the web supervisor will look at the online version of the website and make sure that it is functioning well. If it is, the supervisor will make a copy of the "InProgress" folder and call it "Stable YYYY-MM-DD". This will allow us to revert to a certified working version if necessary.

Step 8: Only the web supervisor should make changes that actually modify the site structure, templates, or the master page(s). Send any such requests and/or bug reports (for example, broken links in the nav-bar) to the supervisor.

 

Documentation Conventions and Tips

Here are some conventions for making pages, as well as some tips for formatting them appropriately.

  • When making a new page, just "duplicate" one of the templates if possible. This will keep them attached to the master page, and it will make sure the formatting is done for you.
  • Be sure to add a title to your page (this is different from giving your page an HTML name). Do so by right clicking on the page, going to "Select" and then "html," and then editing the "title" section of the "properties" tab on the sidebar.
  • Name your page and its corresponding HTML name according to our conventions (which will be updated as we get more information on organization in Pinegrow).
  • 99% of the time (unless you have HTML experience), the edits you need to make will be done on the interface and/or in the toolbar, NOT in the HTML code itself. Editing the code directly could create syntax errors, which will prevent the page from loading properly.
  • If you are creating a new template, give it an HTML name beginning with "aa" so that it is automatically sorted to the top of the pages list.
  • When placing elements on a page, it is easier to drag them to the "Tree," as opposed to dragging them directly to the page. That way, you avoid errors in the HTML code (e.g., accidentally placing a paragraph block inside of another paragraph block).
